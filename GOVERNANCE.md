# Governance

`swift-skills` is maintainer-led.

## Maintainers

Maintainers are responsible for:

- repository direction and scope
- review and acceptance of contributions
- protecting boundary clarity across skills, templates, profiles, and examples

## Decision Making

- Small changes can be proposed directly in pull requests.
- Larger changes that affect skill boundaries, repo governance, or public
  repository shape should be proposed in an issue first.
- Maintainers make final decisions when tradeoffs or scope questions remain.

## Change Expectations

Prefer small, explicit changes. When a change affects multiple layers of the
repo, call out the boundary impact clearly in the issue or pull request.
